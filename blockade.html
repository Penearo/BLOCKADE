Máquina Virtual Lista para Usar

Blockade ha sido diseñado como una solución "Plug & Play" para el usuario final. Para evitar el despliegue manual desde cero, se ha preparado una máquina virtual (archivo .ova) completamente preconfigurada.

Esta máquina ya incluye los adaptadores de red ajustados (NAT e Interna), el firewall IPTables configurado, y todos los servicios descritos en este manual instalados y sincronizados.

02 Visión general

Arquitectura del sistema

Blockade actúa como filtro centralizado a nivel de red. Todo el tráfico del hogar pasa a través del servidor antes de llegar a Internet, donde AdGuard Home bloquea dominios maliciosos, publicidad y contenido inapropiado sin que ningún dispositivo pueda esquivarlo. Ideal para familias preocupadas por el control parental, pero sin conocimientos técnicos avanzados.

03 Instalación del servidor

01
Importar la OVA de Debian 13 en VirtualBox
Abrir VirtualBox → Archivo → Importar servicio virtualizado y seleccionar el archivo .ova de Debian 13 con 1024 MB de RAM.
02
Configurar los dos adaptadores de red
En Configuración de la VM → Red, establecer las dos tarjetas:
AdaptadorModoUso
Adaptador 1 (enp0s3)NATSalida a Internet
Adaptador 2 (enp0s8)Red InternaLAN del hogar (servidor-hijo)
03
Actualizar el sistema e instalar SSH
Bash
# Actualizar lista de paquetes
apt update && apt upgrade -y

# Instalar SSH para trabajar remotamente de forma cómoda
apt install ssh -y
04
Instalar curl y descargar AdGuard Home
Bash
apt install curl -y

# Descargar e instalar AdGuard Home con su script oficial
curl -s -S -L \
  https://raw.githubusercontent.com/AdguardTeam/AdGuardHome/master/scripts/install.sh \
  | sh -s -- -v
05
Asignar IP fija al adaptador de red interna
Editar el archivo de interfaces para fijar la IP 192.168.0.100 en enp0s8:
Bash
nano /etc/network/interfaces

# Añadir al final del archivo:
auto enp0s8
iface enp0s8 inet static
    address 192.168.0.100
    netmask 255.255.255.0
Bash
systemctl restart networking
06
Configurar IP estática en el cliente Windows (Temporal)
Para poder acceder al panel de AdGuard antes de configurar el DHCP, en la máquina Windows 11 configuramos temporalmente la IP manual:
CampoValor de ejemplo
Dirección IP192.168.0.101
Máscara de subred255.255.255.0

Paso 6

06 Configurar el servidor DHCP

Instalamos DHCP para que asigne automáticamente las IPs, la puerta de enlace y el DNS a los clientes de la red interna, permitiendo que salgan a Internet a través de la tarjeta NAT del servidor.

01
Instalar el servidor DHCP
Bash
apt install isc-dhcp-server -y
02
Indicar el adaptador que gestionará el DHCP
Bash
nano /etc/default/isc-dhcp-server

# Modificar esta línea especificando IPv4:
INTERFACESv4="enp0s8"
03
Definir el rango de IPs, la puerta de enlace y el DNS
Bash
nano /etc/dhcp/dhcpd.conf

# Añadir/modificar la configuración principal:
option domain-name "blockade.com";
option domain-name-servers 192.168.0.100;

default-lease-time 600;
max-lease-time 7200;

subnet 192.168.0.0 netmask 255.255.255.0 {
    range 192.168.0.20 192.168.0.40;
    option routers 192.168.0.100;
    option domain-name-servers 192.168.0.100;
}
04
Activar DHCP y comprobar en el cliente
Bash
systemctl restart isc-dhcp-server
systemctl status isc-dhcp-server
En la máquina Windows (CMD), borrar la configuración manual anterior y forzar la nueva concesión:
CMD
ipconfig /release
ipconfig /renew
Paso 7

07 Configurar DNS con BIND9

Vamos a indicarle a AdGuard, que lleva un DNS integrado, que le ceda el puerto estándar a BIND9, el cual se encargará de reenviar las peticiones al motor de filtrado.

01
Instalar BIND9
Bash
apt install bind9 bind9-doc -y
02
Cambiar el puerto DNS de AdGuard al 5454
Bash
systemctl stop AdGuardHome
nano /opt/AdGuardHome/AdGuardHome.yaml

# Asegurarse de mantener la web en el 3000 y cambiar el DNS al 5454:
http:
  address: 0.0.0.0:3000
dns:
  port: 5454
03
Configurar BIND9 para reenviar a AdGuard
Le indicamos a BIND que escuche en nuestra IP, permita consultas de cualquiera y reenvíe el tráfico a Internet al puerto 5454.
Bash
nano /etc/bind/named.conf.options

options {
    directory "/var/cache/bind";
    listen-on { 192.168.0.100; 127.0.0.1; };
    recursion yes;
    allow-query { any; };
    forwarders {
        127.0.0.1 port 5454;
    };
    forward only;
    dnssec-validation no;
};
04
Declarar la zona local blockade.com
Bash
nano /etc/bind/named.conf.local

zone "blockade.com" {
    type master;
    file "/etc/bind/db.blockade.com";
};
05
Crear el archivo de zona directa
Bash
nano /etc/bind/db.blockade.com

$TTL 604800
@   IN  SOA  ns1.blockade.com. admin.blockade.com. (
              2       ; Serial
         604800       ; Refresh
          86400       ; Retry
        2419200       ; Expire
         604800 )     ; Negative Cache TTL

@       IN  NS   ns1.blockade.com.
ns1     IN  A    192.168.0.100
manual  IN  A    192.168.0.100
06
Apuntar el DNS del servidor a sí mismo
Bash
nano /etc/resolv.conf

nameserver 192.168.0.100
Bash
systemctl restart bind9
systemctl start AdGuardHome

Paso 9

09 Forzar el DNS con IPTables

Sin estas reglas, cualquier usuario puede cambiar el DNS en su dispositivo y saltarse todos los filtros. IPTables intercepta cualquier petición DNS y la redirige al servidor obligatoriamente.

01
Activar el reenvío de paquetes IP (IP Forwarding)
Crea un archivo en sysctl para que el Kernel active el enrutamiento y no lo pierda en reinicios:
Bash
echo "net.ipv4.ip_forward=1" > /etc/sysctl.d/99-router.conf
02
Aplicar reglas NAT y redirección DNS obligatoria
Limpiamos reglas anteriores, aplicamos el enmascaramiento para la salida a internet y obligamos al tráfico TCP/UDP por el puerto 53 a usar el servidor local.
Bash
# Borrar reglas previas
iptables -t nat -F

# Salida a internet (NAT)
iptables -t nat -A POSTROUTING -o enp0s3 -j MASQUERADE

# Obligar DNS local (UDP y TCP)
iptables -t nat -A PREROUTING -i enp0s8 -p udp --dport 53 -j REDIRECT --to-port 53
iptables -t nat -A PREROUTING -i enp0s8 -p tcp --dport 53 -j REDIRECT --to-port 53
03
Guardar las reglas de forma persistente
Para que IPTables no olvide la configuración de seguridad al reiniciar el servidor:
Bash
apt install iptables-persistent -y
Al instalar, nos aparecerá una ventana gráfica para confirmar. Seleccionamos <Sí> para guardar las reglas IPv4 actuales.
Infraestructura Blockade completada. Todos los dispositivos de la red interna tienen sus peticiones DNS filtradas con control parental activo y protección contra malware, sin posibilidad de ser evitado desde los clientes.
